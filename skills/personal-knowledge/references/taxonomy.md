# 知识分类

使用下面这些一级分类组织技术知识。

## 分类

### AI (`content/ai/`)

AI 工程、Prompt、RAG、Agent、模型评估、上下文管理、AI 产品模式和模型集成。

例子：

- Prompt 设计
- RAG 切分策略
- Agent 工具调用
- 模型评估
- AI 应用安全

### 架构与系统设计 (`content/architecture/`)

架构、系统设计、方案设计、技术取舍、模块边界、可扩展性、可靠性和演进路径。

例子：

- 模块化单体与微服务
- 登录系统设计
- 知识库架构
- 基于队列的异步设计
- 回滚与降级策略

### 工程实践 (`content/engineering/`)

通用工程实践、测试、调试、代码审查、重构、可维护性、可观测性和交付质量。

例子：

- Code review checklist
- 测试策略
- 调试流程
- 错误处理
- 日志与监控

### 数据库与存储 (`content/database/`)

存储、SQL、NoSQL、事务、索引、缓存、Schema 设计、查询优化和数据一致性。

例子：

- 事务隔离级别
- 索引设计
- 缓存失效
- 数据建模
- 查询性能

### 前端 (`content/frontend/`)

浏览器行为、UI 工程、前端架构、状态管理、渲染、可访问性和前端性能。

例子：

- 组件设计
- 客户端状态与服务端状态
- 渲染性能
- 表单状态
- 可访问性模式

### 后端 (`content/backend/`)

API、服务、认证、授权、队列、Worker、后端架构、服务边界和集成模式。

例子：

- REST API 设计
- Auth flow
- 后台任务
- 服务错误处理
- Rate limiting

### 工具与效率 (`content/tools/`)

开发工具、Git、CI/CD、编辑器工作流、命令行效率、自动化和本地开发环境。

例子：

- Git 工作流
- CI pipeline
- 本地开发脚本
- Shell 效率
- 仓库自动化

### 故障与复盘 (`content/incidents/`)

Bug、故障、复盘、调试记录、生产事故，以及从失败中沉淀的经验。

例子：

- Bug 根因分析
- 故障复盘
- 回归问题调查
- 性能事故
- 预防清单

## 分类规则

- 优先选择最能代表笔记复用价值的分类。
- 如果笔记重点是系统方案选择，放入 `architecture`。
- 如果笔记重点是 AI 系统行为或模型集成，放入 `ai`。
- 如果笔记来自 bug、故障或排查过程，放入 `incidents`。
- 如果笔记重点是代码质量、交付质量或工程方法，放入 `engineering`。
- 如果两个分类都合适，选择一个主分类，后续可以从相关分类索引补链接。
